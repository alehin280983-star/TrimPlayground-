## 🚫 ЗАПРЕЩЕНО

### Категорически нельзя:

1. **Создавать файлы вне списка разрешённых** (см. раздел "Разрешённые файлы")
2. **Изменять файлы, не указанные в текущей задаче**
3. **Добавлять новые npm зависимости** без явного указания в задаче
4. **Менять структуру папок проекта**
5. **Удалять существующий код** без явного указания
6. **Создавать новые API endpoints** (только улучшать существующие)
7. **Менять конфигурацию моделей** в `src/lib/config/models.ts`
8. **Трогать файлы в** `node_modules/`, `.next/`, `backups/`

## 🔒 ПРАВИЛА КОДА

### TypeScript

1. **Всегда** использовать типы из `src/types/`
2. **Никогда** не использовать `any` — только `unknown` с type guards
3. **Экспортировать** только через `index.ts`

### React Components

1. **'use client'** — обязательно для интерактивных компонентов
2. **Props interface** — всегда определять явно
3. **CSS** — использовать только Tailwind классы из существующего дизайна
4. **Naming** — PascalCase для компонентов, camelCase для функций

### API Routes

1. **Валидация** — всегда проверять входные данные
2. **Ошибки** — возвращать в формате `{ success: false, error: { code, message } }`
3. **Timeout** — использовать `executeWithTimeout` для внешних вызовов

## 🧪 ТЕСТИРОВАНИЕ

### Перед завершением задачи:

1. **Компиляция**: `npm run build` — без ошибок
2. **Lint**: `npm run lint` — без ошибок  
3. **Dev-сервер**: `npm run dev` — страница открывается без ошибок в консоли

### Если задача касается UI:

1. Открыть `http://localhost:3000/playground`
2. Проверить визуально
3. Проверить в DevTools (нет ошибок)
